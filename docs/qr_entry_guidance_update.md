# QRコード導線優先改修 仕様書

作成日: 2025-10-27  
対象: 生駒祭 屋台ランキング評価フォーム（ikomasai.com）

---

## 1. 背景

- 現行トップページは屋台番号入力フォームが主導線となっており、来場者がまず番号入力を求められる UI になっている。
- 実際の運用では各屋台に掲出した QR コードからの誘導がメインであり、トップページに直接アクセスする来場者も QR 誘導を受ける前提で案内したい。
- QR コードを読み取れないケースに備えて、既存の屋台番号入力導線を残しつつもサブ導線として扱いたい。

## 2. 目的

- 直接アクセス時に「QRコードを読み取ってください」というメッセージを主導線として提示し、現地での案内内容と一致させる。
- QR を読み取れない・読み取り方がわからない来場者向けに、既存の屋台番号入力フォームへ遷移できる補助導線を提供する。

- 既存の QR 誘導（`?booth=***` 付き URL）でアクセスした場合は、従来通りすぐに評価フォームへ進む体験を維持する。

## 3. スコープ

- 対象ページ: `index.html`（ikomasai.com のトップページ）
- 関連スタイル: `style.css`
- 動作制御: `main.js`
- バックエンド・Supabase 連携・Google フォーム送信仕様には変更なし。

## 4. ユーザーフロー変更

### 4.1 初期表示

| 条件 | 期待表示 |
|------|----------|
| `booth` クエリパラメータあり（例: `?booth=A01`） | 既存挙動を維持。屋台番号ゲートは非表示のまま評価フォームへ遷移。 |
| `booth` クエリパラメータなし | 新しい「QRコード案内セクション」を hero 直下に表示し、屋台番号ゲートは初期状態で折りたたむ。 |

### 4.2 手動入力導線

1. 「QRコードを読み取れない方はこちら」ボタン（またはリンク）をタップ/クリック。
2. ボタン押下で屋台番号ゲート `#booth-gate` の `hidden` クラスを解除し、フォーカスを入力欄 `#gate-booth` に移動する。
3. 同時に案内セクション文言は画面上に残す（閉じない）。屋台番号ゲートはサブ導線としてスクロール内に展開する。

### 4.3 既存ロジックとの整合

- ローカルストレージによる重複投票ブロック、評価フォーム表示制御、送信フローは変更しない。
- 手動入力でフォームに進む際も既存の `handleGateSubmit` 処理をそのまま利用する。

## 5. UI / コピー仕様

### 5.1 QR案内セクション構成案

- 見出し: `屋台のQRコードを読み取って評価ページへ`
- 本文: `各屋台に掲示されているQRコードからアクセスすると、該当屋台の評価フォームがすぐに開きます。まずはQRコードを読み取ってください。`
- CTA（サブ導線）: ボタンテキスト `QRコードを読み取れない方はこちら`
- レイアウト: 既存カードデザインを踏襲し、`card` クラスを流用。ヒーロー下に配置。

### 5.2 アクセシビリティ

- サブ導線ボタンは `<button type="button">` もしくは `<a role="button">` で実装し、キーボード操作可能にする。
- ボタン押下時のフォーカス移動は `focus()` を使用し、スクリーンリーダー利用者にもゲート表示が伝わるよう状態変更時に `aria-expanded="true"` を設定。
- `#booth-gate` を非表示にする際は `hidden` クラス＋`aria-hidden="true"` を追加し、表示時に両方を外す。

### 5.3 レスポンシブ

- スマートフォン表示を前提に、案内セクションのテキストサイズと行間を調整（`16px/1.6` 程度）。
- サブ導線ボタンはフル幅（`width: 100%`）で配置し、タップ領域を確保。
- PC/タブレットでは従来のレイアウト幅 (`max-width: 960px`) に合わせる。

## 6. 実装方針

### 6.1 HTML（`index.html`）

- Hero セクション直下に新規セクション `<section id="qr-notice" class="card notice">`（仮称）を追加。
- `#booth-gate` セクションには初期状態で `hidden` クラスと `aria-hidden="true"` を付与。
- `#qr-notice` 内にサブ導線ボタンを配置し、`data-target="booth-gate"` などで JavaScript と連携する。

### 6.2 CSS（`style.css`）

- `card` のバリエーションとして `.card.notice` 向けに余白・背景の微調整を追加。
- `#booth-gate.hidden` 用のスタイル（`display: none;`）を既存の `hidden` クラスに合わせて定義済みか確認し、不足していれば追加。
- サブ導線ボタンを `.btn btn--outline` など視認性のあるスタイルで定義。

### 6.3 JavaScript（`main.js`）

- DOMContentLoaded 後に以下を判定：
  - `booth` クエリがある場合：`#qr-notice` を非表示（任意）、`#booth-gate` は表示させず従来フロー。
  - ない場合：`#booth-gate` を閉じたまま、サブ導線ボタンにクリックイベントを設定。
- サブ導線イベントハンドラ：
  - `#booth-gate` から `hidden` クラスと `aria-hidden` を外す。
  - `scrollIntoView({ behavior: 'smooth' })` でゲートまでスクロール（任意）。
  - 入力欄へ `focus()` を付与。
  - 2回目以降押下では何も起きないようガード。
- 既存の `showGateSectionIfNeeded` 等の関数があれば再利用し、無ければ小規模関数を追加。

## 7. 非機能要件 / 配慮事項

- Lighthouse スコアや OG タグ等の SEO 設定は従来維持。
- 表示文言は日本語のみ。必要に応じて多言語化余地をコメントで残す。
- ローカルストレージや Supabase API への影響はないことを確認する。

## 8. テスト観点

- [ ] `?booth=A01` でアクセスした際に案内セクションを経由せず、フォームが正しく表示される。
- [ ] クエリなしでアクセス → 案内セクションが表示され、屋台番号ゲートが閉じている。
- [ ] サブ導線ボタン押下でゲートが展開し、入力可能になる。
- [ ] ゲート展開後に従来通り屋台番号バリデーション・エラー表示が動作する。
- [ ] スマホ・PC それぞれでレイアウト崩れがない。
- [ ] キーボード操作のみでサブ導線→屋台番号入力→送信まで到達できる。

## 9. 移行・公開手順

1. ローカルでレイアウトと挙動を確認。
2. 必要に応じて QR コードに印刷されている URL が `?booth=` 付きであることを再確認。
3. 本番デプロイ前にキャッシュクリア（Cloudflare 等）手順を確認。
4. 公開後 1～2 日はアクセスログを確認し、直アクセスユーザーの離脱がないか注視。
